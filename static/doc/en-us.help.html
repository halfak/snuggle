Snuggle version: 1.0rc1

Quick Reference

See the full documentation

Snuggle is designed to help you identify desirable new Wikipedia editors. The interface lists all accounts registered in the last 30 days along with contributions and talk page messages.

User interface

Figure 1. Controls.
  1. Hand curated lists of users
    • uncategorized - New users that have not been categorized yet.
    • good-faith - New users that are at least trying to do something productive.
    • ambiguous - New users whose motivations are unclear.
    • bad-faith - New users that do not appear to be trying to be productive.
  2. Minimum number of edits.
  3. The namespace that the minimum number of edits must exist in.
  4. Metrics by which to sort users
    • desirability - A machine-generated probability that the user is editing in good faith.
    • registration date - The date that user registered.
    • last activity - The date of the user's last edit.
    • reverted edits - The number of the users edits that were reverted.
    • total edits - The total number of edits the user performed.
  5. The direction to sort the users. This allows you to reverse the order of any of the sorting metrics.
Figure 2. Expanded user.
  1. User account metadata and statistics.
    • Username
    • User - Talk - Contribs
    • Registration date
    • Last edit date
    • # of times viewed in Snuggle
    • Revision count graph
    • # of time reverted (non-self)
  2. Talk page activity. Certain types of messages are leveled with an icon e.g.:
    • Proposal for deletion
    • level 2 warning
    • welcome message
    • speed deletion
    • General discussion
  3. Categorization buttons. This will move the user into one of the category lists (discussed below).
  4. Interactive graph of user contribs for the first 30 days. Different colors represent different namespaces. E.g.:
    • Article
    • Talk
    • User
    • User_talk
    • Wikipedia
    • Wikipedia_talk
    Revisions reverted by another editor are noted with a dot (). Self reverts have a hollow dot ()

Keyboard shortcuts

While you can use the mouse to peform all actions, you can also use a set of keyboard shortcuts to peform some common actions at high speed.

User selection
Page Up moves up and Page Down moves down
Revision selection (in the graph)
The arrow keys: ↑, →, ↓ and ←
Categorize the selected user
Number keys: #1 for "good-faith", #2 for "ambiguous" and #3 for "bad-faith"
